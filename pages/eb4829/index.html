个人常用Git命令指南

# 个人常用Git命令指南

建议

不要越过命令行这座山而直接去使用 GUI。先不论 GUI 是否支持复杂的操作，当在实际工作中遇到问题的时候，可能无法找到 GUI 的解决方案，因为搜索引擎查出来的都是命令行。而且命令行的错误提示是非常详细的，可以准确锁定错误的原因以及正确的使用方法，孰能自然生巧。

整理了一些个人常用的 Git 操作，方便工作查阅及温故知新。

# 配置操作

# 配置全局git用户名
git config --global user.name [用户名]

# 配置全局git邮箱
git config --global user.email [邮箱]

# 查看全局配置
git config --global --list

# 本地操作

# 查看文件、文件夹在工作区、暂存区的状态
git status

# 将文件加入到暂存区中，实现追踪
git add ./[文件名]

# 将修改的内容暂存，并让本地代码回退到修改前的状态
git stash

# 将暂存的内容恢复
git stash apply

# 比较工作区和暂存区的所有差异
git diff

# 将文件从暂存区中撤销出来
git reset HEAD [文件名]

# 项目回到上一个版本
git reset --hard HEAD^

# 项目回到指定的版本
git reset --hard [提交ID]

# 指定文件回到指定的版本
git checkout [提交ID] -- [文件名]

# 分支操作

# 创建分支
git branch [分支名]

# 查看本地所有分支
git branch

# 查看本地分支与远程分支的关系
git branch -av

# 切换到指定的分支
git checkout [分支名]

# 删除指定的分支
git branch -d [分支名]

# 创建并切换到指定的分支
git checkout -b [分支名]

# 创建切换到本地分支，并与远程分支做关联
git checkout -b [本地分支] [远程分支]

# 将此分支合并到当前的分支
git merge [分支名]

# 合并冲突时，忽略合并过来的分支代码，保留原分支代码
git merge --abort

# 删除远程分支
git push origin --delete [分支名]

# 变更历史

# 查看全部提交历史的详细信息
git log

# 查看全部提交历史的简要信息
git log --oneline

# 查看版本路线
git log --oneline --graph

# 查看某个用户的提交历史
git log --author=[用户名]

# 查看本地所有历史记录
git reflog

# 标签操作

# 给最新的一次提交创建标签
git tag [标签名]

# 查看所有标签
git tag

# 给指定id的提交创建标签
git tag [标签名] [提交版本ID]

# 删除指定的标签
git tag -d [标签名]

# 将指定的标签推送到远程
git push origin [标签名]

# 远端交互

# 查看关联的所有远程仓库名称及地址
git remote -v

# 与远程仓库做关联
git remote add origin [远程仓库地址]

# 从远程获取最新版本到本地，但不会自动合并（取回的代码对你本地的开发代码没有影响）
git fetch

# 从远程拉取最新版本到本地并自动merge本地代码
git pull

# 提交本地仓库到远程（-u 是设置默认的推送地址，下次可以直接使用git push命令）
git push -u origin master
